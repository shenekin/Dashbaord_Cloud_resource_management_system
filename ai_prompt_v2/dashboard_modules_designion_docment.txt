# 一、①《Dashboard 前端页面模块设计文档》（正式版）
> 文档定位：
> **云资源管理平台 · Dashboard 页面信息架构 & 模块设计说明**
## 1. 页面基本信息
* **Page Name**：Dashboard
* **Page Type**：Global Overview / Control Center
* **Target Users**：
  * Platform Admin
  * Ops / SRE
  * Project Owner（裁剪后）
## 2. 页面整体布局结构（Layout）

Dashboard Layout
├── Global Header（全局顶部）
├── Sidebar Navigation（左侧导航）
└── Main Content Area（主内容区）

### 布局原则
* Header、Sidebar **全局复用**
* Main Content **页面级独立**
* 支持大屏 / 宽屏（运维常见）
## 3. Global Header 模块设计
### 3.1 模块职责
| 子模块               | 职责说明        |
| ----------------- | ----------- |
| Brand Area        | 产品标识 / Logo |
| Global Search     | 全平台资源搜索     |
| System Indicators | 平台健康状态      |
| User Control      | 用户与身份操作     |

### 3.2 System Indicators（重点）
展示内容：
* Gateway 状态（Normal / Warning / Down）
* Active Alerts 数量
* Pending Tasks / Approvals 数量
**设计目的**：
> 用户打开平台第一眼就知道：
> **“平台现在稳不稳”**
## 4. Sidebar Navigation 模块设计
### 4.1 Sidebar 功能定位
> Sidebar = **平台能力地图**
### 4.2 导航结构定义

Dashboard
Gateway
User & Auth
Projects
Resources
  ├── ECS
  ├── Network
  ├── Images / Storage
  └── Security Groups
Quotas
Tasks
Monitoring
Billing
Audit Logs
Notifications

### 4.3 Sidebar 行为规则
* 当前模块高亮
* 二级菜单可折叠
* 支持权限控制（不可见 / 不可点）
## 5. Main Content 模块拆分（核心）
### 5.1 状态摘要区（第一行）
**模块名**：Status Summary Cards
**包含卡片**：
* System Health
* Active Alerts
* Pending Approvals
#### 单卡片结构规范
| 字段          | 说明                        |
| ----------- | ------------------------- |
| Icon        | 状态型图标                     |
| Title       | 模块名称                      |
| Main Value  | 核心数字                      |
| Description | 简要状态说明                    |
| Status      | success / warning / error |

### 5.2 API Gateway Overview
**模块定位**：流量与接入层监控
* 图表类型：折线图
* 核心指标：Requests Per Second
* 维度：时间

### 5.3 User & Role Management
拆分为三个子模块：
1. **User Summary**
   * Total Users
   * Active Users
2. **User Activity Trend**
   * 时间段内活跃度
3. **Role Distribution**
   * Admin / User 比例

### 5.4 ECS Instance Status
**模块定位**：计算资源健康度
* Running / Stopped 数量
* 可用率 / 异常率
* CPU 使用率
* 快捷入口：Manage Instances

### 5.5 Project Resource Usage
**模块定位**：项目维度资源消耗
| 字段      | 含义      |
| ------- | ------- |
| Project | 项目 / 客户 |
| CPU     | 使用率     |
| RAM     | 使用率     |
| Storage | 使用率     |
* 使用进度条表达
* 支持点击进入 Project Detail
### 5.6 Active Alarms
* 告警级别：Critical / Warning / Alert
* 状态：Open / Acknowledged / Resolved
* 快捷操作入口

### 5.7 Task Automation
* Scheduled Tasks 总数
* 执行成功率
* 任务类型：
  * Backup
  * Scaling
  * Maintenance

### 5.8 Cost Overview
拆分为：
1. **Cost Summary**
   * 本月费用
   * 预测费用
2. **Cost Breakdown**
   * 按资源 / 项目拆分
### 5.9 Audit Logs
* Time
* User
* Action
* Details
**用途**：安全 / 合规 / 追责
### 5.10 Recent Notifications
* 通知渠道：Email / SMS / WeChat
* 通知类型：Alert / Approval / Info
# 二、② 后端 API 设计反推文档（不写代码）
> 目标：
> **前端只关心“我要什么数据”，后端自由实现**
## 1. Dashboard API 总体原则

### 1.1 聚合原则（非常重要）
> Dashboard ≠ 微服务直连
> Dashboard = **聚合 + 只读**
* 减少前端调用次数
* 降低跨服务复杂度
* 支持缓存
## 2. API 逻辑分组设计
/dashboard
├── /system
├── /gateway
├── /users
├── /resources
├── /projects
├── /alerts
├── /automation
├── /cost
├── /audit
└── /notifications


## 3. 模块 → API 职责映射表

| 页面模块              | API 提供内容    |
| ----------------- | ----------- |
| System Health     | 服务状态 / 健康检查 |
| Active Alerts     | 未解决告警数量     |
| Pending Approvals | 待审批任务       |
| Gateway Overview  | RPS / 流量趋势  |
| User Summary      | 用户总数 / 活跃数  |
| Role Distribution | 角色占比        |
| ECS Status        | 实例状态统计      |
| Project Resource  | 项目资源使用      |
| Active Alarms     | 告警列表        |
| Automation        | 任务统计        |
| Cost Overview     | 成本汇总 / 预测   |
| Audit Logs        | 操作日志        |
| Notifications     | 最近通知        |

## 4. Dashboard API 设计约束（给你用）
* **只读接口**
* **支持缓存（秒级 / 分钟级）**
* **按角色返回裁剪数据**
* **禁止返回明细级大列表**
## 5. 强烈建议的后端实现模式（非代码）
* 单独 Dashboard Service
* 或 Gateway 层聚合
* 不直接暴露底层服务
